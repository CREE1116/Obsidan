## GPU

### GPU의 구조와 기능

1. 병렬처리 
	- 딥러닝 모델은 대규모의 수학적 연산을 필요로 하며, GPU는 수많은 코어를 통해 이러한 연산을 병렬로 처리.
	- 특히 [[합성곱 신경망]](CNN)에서 각 필터가 독립적으로 연산되므로 병렬처리가 큰 장점을 발휘
2. 전용 하드웨어
	- GPU는 벡터와 행렬연산에 특화된 하드웨어를 가지고 있어 모델의 학습속도를 크게 향상시킴
3. 고성능 메모리
	- GDDR, HBM등을 통해 대용량 데이터와 모델 파라미터를 빠르게 접근하고 처리

### CPU, GPU, NPU란?

1. cpu
	- 컴퓨터의 핵심 연산과 제어를 담당하는 반도체로서 요즘은 Multi-Core(다수의 ALU를 탑재) 형태로 진화중
	- 컴퓨터 내 메모리, 캐시메모리 등을 관리하는 운영체제와 응용프로그램을 실행하는 기능
2. gpu
	- 동영상 및 게임과 같은 응용에서 고성능 그래픽 처리에 유용한 반도체로, 요즘에는 AI의 알고리즘 처리에도 많이 활용됨
	- 그래픽처리를 효율적으로 수행하고자, 다수의 ALU를 화면 픽셀에 매칭시켜, 픽섹별 연산을 빠르게 수행
3. npu
	- 신경망 처리에 유용한 구조로 설계된 반도체로서, GPU보다 인공지능학습에 더 성능이 좋은 구조임

## 컴퓨터 아키텍쳐 과목소개 요약 (상세, MD 형식)

**주요 학습 내용 상세 설명:**

- **컴퓨터 시스템의 Layers: Application/Firmware/OS/Hardware**
    
    - 컴퓨터 시스템은 복잡성을 관리하기 위해 계층 구조로 구성됩니다. 각 계층은 특정 역할을 담당하며, 상위 계층은 하위 계층의 서비스를 이용합니다.
        
        - **Hardware (하드웨어):** 가장 기본적인 계층으로, CPU, 메모리, 저장 장치, 입출력 장치 등 물리적인 부품들을 포함합니다. 컴퓨터 시스템의 기반이 됩니다.
            
        - **OS (Operating System, 운영체제):** 하드웨어 자원을 관리하고, 응용 프로그램이 하드웨어를 효율적으로 사용할 수 있도록 인터페이스를 제공하는 소프트웨어입니다. 파일 시스템 관리, 프로세스 관리, 메모리 관리 등을 담당합니다.
            
        - **Firmware (펌웨어):** 하드웨어에 내장된 기본적인 소프트웨어로, 하드웨어를 초기화하고 기본적인 기능을 제공합니다. BIOS나 임베디드 시스템의 초기 부팅 코드 등이 펌웨어에 해당합니다. (강의 자료에서는 Firmware가 명시적으로 강조되지는 않았지만, 시스템 계층의 일부로 이해할 수 있습니다.)
            
        - **Application (응용 프로그램):** 사용자가 직접 사용하는 소프트웨어입니다. 워드 프로세서, 웹 브라우저, 게임 등 다양한 종류가 있으며, OS의 서비스를 이용하여 특정 작업을 수행합니다.
            
    - 이 계층 구조를 이해하는 것은 컴퓨터 시스템의 작동 방식을 전체적으로 파악하는 데 중요합니다.
        
- **Machine Code vs. Assembly Language**
    
    - **Machine Code (기계어):** CPU가 직접 이해하고 실행할 수 있는 유일한 언어입니다. 2진수(0과 1)로만 구성되어 있으며, CPU 종류(아키텍쳐)마다 형식이 다릅니다.
        
        - **특징:** 2진수 코드, CPU 의존적, 매우 낮은 수준의 언어
            
        - **단점:** 인간이 읽고 이해하기 매우 어렵고, 프로그래밍 생산성이 낮습니다.
            
    - **Assembly Language (어셈블리어):** 기계어를 사람이 읽기 쉽도록 기호화한 언어입니다. 각 기계어 명령어에 대해 니모닉(mnemonic, 연상기호, 예: ADD, SUB, MOV)을 사용합니다.
        
        - **특징:** Low-level 프로그래밍 언어, CPU 의존적, 기계어와 일대일 대응
            
        - **장점:** 기계어보다는 인간이 읽고 이해하기 쉽고, 하드웨어 제어가 용이합니다.
            
    - 어셈블리어는 기계어와 밀접하게 관련되어 있어, 컴퓨터 하드웨어의 작동 원리를 깊이 이해하는 데 도움이 됩니다.
        
- **High-level 언어 vs. Low-level 언어**
    
    - **High-level Language (고급 언어):** C, C++, Java, Python 등과 같이 인간이 이해하고 작성하기 쉬운 언어입니다. 문법이 자연어에 가깝고, 추상화 수준이 높아 복잡한 프로그램을 비교적 쉽게 개발할 수 있습니다.
        
        - **특징:** 인간 친화적, 추상화 수준 높음, 다양한 플랫폼에서 사용 가능 (이식성)
            
        - **컴파일러 (Compiler):** 고급 언어로 작성된 소스 코드를 기계어로 번역하는 프로그램입니다. 컴파일 과정을 통해 실행 가능한 프로그램을 생성합니다.
            
    - **Low-level Language (저급 언어):** 어셈블리어와 기계어를 포함하며, 하드웨어에 가까운 언어입니다. 하드웨어 제어가 필요한 시스템 프로그래밍이나 임베디드 시스템 개발에 주로 사용됩니다.
        
        - **특징:** 하드웨어 제어 용이, 성능 최적화 가능, CPU 의존적
            
        - **어셈블러 (Assembler):** 어셈블리어로 작성된 코드를 기계어로 번역하는 프로그램입니다.
            
    - 고급 언어와 저급 언어는 각각 장단점이 있으며, 개발 목적과 환경에 따라 적절한 언어를 선택해야 합니다. 컴퓨터 아키텍쳐를 배우면서는 저급 언어와 하드웨어의 관계를 이해하는 것이 중요합니다.
        
- **컴파일 프로세스**
    
    - 고급 언어로 작성된 소스 코드가 실행 가능한 프로그램으로 변환되는 과정을 컴파일 프로세스라고 합니다. 일반적으로 다음과 같은 단계로 구성됩니다.
        
        - **Pre-Processor (전처리기):** 소스 코드에서 전처리 지시어(# include, # define 등)를 처리합니다. 헤더 파일을 포함시키거나 매크로를 치환하는 등의 작업을 수행합니다.
            
        - **Compiler (컴파일러):** 전처리된 소스 코드를 어셈블리어 코드로 변환합니다. 문법 분석, 의미 분석, 최적화 등의 과정을 거칩니다.
            
        - **Assembler (어셈블러):** 컴파일러가 생성한 어셈블리어 코드를 기계어 코드 (Object file, 목적 파일)로 변환합니다.
            
        - **Linker (링커):** 여러 개의 목적 파일과 라이브러리 (미리 컴파일된 함수들의 모음)를 결합하여 최종 실행 파일 (Executable file)을 생성합니다. 링킹 과정에서 서로 다른 목적 파일 간의 참조를 연결하고, 필요한 라이브러리 코드를 포함시킵니다.
            
    - 컴파일 프로세스를 이해하는 것은 프로그램 개발 과정과 실행 원리를 파악하는 데 필수적입니다.
        
- **임베디드 시스템 vs. 일반 컴퓨터 시스템**
    
    - **Embedded/Dedicated System (임베디드/전용 시스템):** 특정 목적이나 기능을 수행하도록 설계된 컴퓨터 시스템입니다. 스마트폰, 냉장고, 자동차 ECU, IoT 기기 등 다양한 전자기기 내부에 탑재됩니다.
        
        - **특징:** 특정 기능에 최적화된 Hardware/Software 설계, 실시간성, 저전력 소비, 소형화 등이 중요합니다.
            
        - **예시:** 스마트폰, 로봇, 자동차 ECU, 통신기기, 의료기기, 산업 제어 시스템 등
            
    - **General-Purpose Computer System (일반 목적 컴퓨터 시스템):** 다양한 용도로 사용할 수 있도록 설계된 컴퓨터 시스템입니다. 데스크톱, 노트북, 서버 등이 해당됩니다.
        
        - **특징:** 표준화된 Hardware 활용, 범용성, 확장성, 사용자 편의성이 중요합니다.
            
        - **예시:** 데스크톱 PC, 노트북, 서버, 메인프레임 컴퓨터 등
            
    - 임베디드 시스템과 일반 컴퓨터 시스템은 설계 목표와 특징이 다르며, 컴퓨터 아키텍쳐 연구에서도 서로 다른 측면을 고려해야 합니다.
        
- **하드웨어 엔지니어 역할 변화**
    
    - 과거에는 기업 내에서 제품/프로젝트별로 독자적인 하드웨어 설계와 생산을 진행하는 경우가 많았습니다. Chip-level의 하드웨어 설계 팀이 존재했습니다.
        
    - 현재는 대부분의 하드웨어 설계와 생산을 전문 반도체 회사에 **아웃소싱**하는 추세입니다. 반도체 회사에서 개발용 보드를 제공하고, 기업은 필요한 설계 변경을 아웃소싱 받아 추진하는 형태로 변화했습니다.
        
    - 결과적으로 일반 기업 내에서 하드웨어 설계 업무는 크게 줄어들고, 아웃소싱 관리를 담당하는 역할이 중요해졌습니다.
        
    - 하지만 **실시간 제어가 필요한 임베디드 시스템** 분야에서는 여전히 하드웨어에 대한 깊이 있는 이해가 중요합니다. 소프트웨어 개발자도 실시간 OS, 하드웨어 지식 등을 갖추는 것이 요구되고 있습니다.
